1. 상속
상속이랑 객체 단위 코드를 재사용하는 방법
기능적 확장이나 개선의미
상속을 이햏나는데 가장 중요한 것은 관계

미래를 위해 설계
나를 위한 설계가 아닌 남을 위한 설계

class 파생클래스이름 : 접근제어지시자 기본 클래스 이름

//기본 클래스 혹은 부모 클래스
class Parent {
...
}

//파생 클래스 혹은 자식 클래스
class child :public Parent 
{
..
}

이러한 파생 관계는 is-a 관계, has-a 관계로 설명 가능

파생된 클래스는 기본 클래스의 멤버에 접근 기능하지만 private 멤버에는 접근 불가

사용자코드에서 파생클래스의 인스턴스를 통해 기본 클래스의 매서드를 호출 가능
